## 版本说明

### 完整版

包含所有组件，适合需要完整功能和企业级部署。

| 组件 | 说明 |
|------|------|
| PostgreSQL | 数据库 |
| Redis | 缓存（高并发场景） |
| RabbitMQ | 消息队列（异步转换） |
| Nginx | Web 服务器 |
| DocService | 核心编辑服务 |
| FileConverter | 文档转换服务 |
| Metrics | 监控服务 |
| AdminPanel | Web 管理界面 |

**估计镜像大小**：~2.5GB

### API 精简版

仅保留核心组件，适合纯 API 集成场景。

| 组件 | 说明 |
|------|------|
| PostgreSQL | 数据库 |
| Nginx | Web 服务器 |
| DocService | 核心编辑服务 |
| FileConverter | 文档转换服务 |

**已移除**：Redis、RabbitMQ、Metrics、AdminPanel、MSSQL/MySQL 客户端

**估计镜像大小**：~1.5GB

## 完整编译流程

### 步骤一：构建编译环境镜像

```bash
docker build -f Dockerfile.build -t onlyoffice-build:latest .
```

### 步骤二：启动编译容器

```bash
docker run -d --name onlyoffice-build \
  -v onlyoffice-build-data:/build_output \
  onlyoffice-build:latest
```

### 步骤三：查看编译进度

```bash
docker logs -f onlyoffice-build
```

编译过程会依次安装：
1. Python 环境
2. Qt 预编译二进制
3. 系统依赖
4. CMake
5. Sysroot
6. 核心组件（V8、ICU 等）
7. 文档服务组件
8. 服务器组件

### 步骤四：修改源码（可选）

编译完成后，如需修改源码：

```bash
# 进入编译容器
docker exec -it onlyoffice-build bash

# 执行源码修改
cd /build_tools
./scripts/patch_source.sh

# 重新编译（可选）
cd tools/linux
./automate.py server --update=0
```

`patch_source.sh` 会进行以下修改：
- 连接数改为 2000
- 开启 advancedApi
- 添加 historyTurnOff/historyTurnOn 方法
- 使用 QT 镜像地址
- 关闭源码更新

### 步骤五：打包完整版镜像

```bash
# 复制编译产物
docker cp onlyoffice-build:/build_output/onlyoffice ./output/package/

# 构建镜像
docker build -f Dockerfile.documentserver \
  -t onlyoffice-documentserver:custom \
  ./output/package/
```

### 步骤六：打包 API 精简版镜像

```bash
# 构建
./build_and_package_api.sh

# 或手动构建
docker build -f Dockerfile.documentserver-api \
  -t onlyoffice-documentserver:api \
  ./output/package-api/
```

## API 精简版说明

### 适用场景

- 仅使用 DocumentServer API
- 通过上层反向代理处理 HTTPS
- 资源受限环境
- Kubernetes/容器编排部署

### 启动脚本自动检测

`run-document-server-api.sh` 会在启动时自动检测 SSL 证书：

| 检测结果 | 行为 |
|----------|------|
| 有证书 | 启用 HTTPS (443端口) |
| 无证书 | 使用 HTTP (80端口) |

## Docker 部署

### 使用 docker-compose

#### 完整版 + API 版

```bash
docker-compose up -d
```

#### 仅 API 精简版

```yaml
# docker-compose.yml
version: '3.8'

services:
  documentserver-api:
    image: onlyoffice-documentserver:api
    container_name: onlyoffice-api
    ports:
      - "8080:80"
      - "8443:443"
    volumes:
      - ./data:/var/www/onlyoffice/Data
      - ./logs:/var/log/onlyoffice
      - ./postgres:/var/lib/postgresql
      - ./fonts:/usr/share/fonts/truetype/custom
      - ./ssl:/etc/letsencrypt
    restart: unless-stopped
```

### 直接运行

```bash
# 无 SSL
docker run -d \
  --name onlyoffice-api \
  -p 8080:80 \
  -v $(pwd)/data:/var/www/onlyoffice/Data \
  onlyoffice-documentserver:api

# 带 SSL
docker run -d \
  --name onlyoffice-api \
  -p 8080:80 \
  -p 8443:443 \
  -v $(pwd)/ssl:/etc/letsencrypt \
  onlyoffice-documentserver:api
```

### 数据持久化

| Volume | 路径 | 说明 |
|--------|------|------|
| documentserver-data | /var/www/onlyoffice/Data | 文档数据 |
| documentserver-logs | /var/log/onlyoffice | 日志文件 |
| documentserver-postgres | /var/lib/postgresql | 数据库 |
| documentserver-fonts | /usr/share/fonts/truetype/custom | 自定义字体 |
| documentserver-ssl | /etc/letsencrypt | SSL 证书 |

## SSL 证书配置

### 证书文件要求

在宿主机创建以下目录结构：

```
ssl/
└── live/
    └── onlyoffice/
        ├── fullchain.pem   # SSL 证书
        ├── privkey.pem     # 私钥
        └── chain.pem       # 证书链（可选）
```

### 获取免费证书

#### 使用 Certbot

```bash
# 安装 Certbot
apt-get install certbot

# 申请证书（需要域名已解析）
certbot certonly --standalone -d your-domain.com

# 证书会保存在
# /etc/letsencrypt/live/your-domain.com/
```

#### 使用 acme.sh

```bash
# 安装 acme.sh
curl https://get.acme.sh | sh

# 申请证书
~/.acme.sh/acme.sh --issue -d your-domain.com --standalone

# 复制到目标位置
~/.acme.sh/acme.sh --install-cert -d your-domain.com \
  --key-file /path/to/ssl/live/onlyoffice/privkey.pem \
  --fullchain-file /path/to/ssl/live/onlyoffice/fullchain.pem
```

### 自动续期

证书有效期 90 天，建议设置自动续期：

```bash
# 添加到 crontab
crontab -e

# 每天检查续期
0 0 * * * ~/.acme.sh/acme.sh --cron --home /root/.acme.sh
```

## 常见问题

### 编译失败

**Q：编译过程中断怎么办？**

```bash
# 重新进入容器继续
docker exec -it onlyoffice-build bash
cd /build_tools/tools/linux
./automate.py server --update=0
```

**Q：Qt 下载失败？**

修改 `tools/linux/automate.py` 中的下载地址为镜像：

```python
# 替换为国内镜像
base.download("https://mirrors.ustc.edu.cn/qtproject/...", "./qt_source_5.9.9.tar.xz")
```

### 容器启动失败

**Q：PostgreSQL 启动失败？**

```bash
# 进入容器排查
docker run -it onlyoffice-documentserver:api /bin/bash

# 手动启动服务
service postgresql start
cat /var/log/postgresql/postgresql-*.log
```

**Q：端口被占用？**

```bash
# 查看端口占用
netstat -tlnp | grep 80

# 使用其他端口
docker run -d -p 8080:80 onlyoffice-documentserver:api
```

### 功能异常

**Q：文档无法打开？**

```bash
# 检查字体
docker exec -it onlyoffice-api ls /usr/share/fonts/truetype/custom/

# 重新编译字体
docker exec -it onlyoffice-api \
  /app/onlyoffice/documentserver/server/tools/allfontsgen \
  --input=/app/onlyoffice/documentserver/core-fonts \
  --allfonts=/app/onlyoffice/documentserver/sdkjs/common/AllFonts.js
```

**Q：转换失败？**

```bash
# 检查转换服务日志
docker exec -it onlyoffice-api tail -f \
  /var/log/onlyoffice/documentserver/converter.log

# 检查文件大小限制
# 默认 32m，在 nginx 配置中 client_max_body_size
```

### 性能问题

**Q：转换速度慢？**

- 增加容器 CPU 限制
- 使用 Redis 缓存（完整版）
- 增加字体编译缓存

**Q：内存不足？**

```yaml
# docker-compose.yml
services:
  documentserver-api:
    deploy:
      resources:
        limits:
          memory: 4G
```
